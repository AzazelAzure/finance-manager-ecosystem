# Audits

Operational, governance, and process audit artifacts. Distinct from:
- `strategy/anomalies/` — point-in-time *detected* issues from agents mid-work.
- `strategy/security/` — automated *security* audit reports (local audit suite output).
- `strategy/risk_register.md` — *anticipated* risks that have not yet occurred.

Audits here are deliberate, scoped reviews of how the operation runs (process, admin, ops
hygiene) and the tracker of improvements coming out of them.

## Contents

| File | Purpose |
|---|---|
| `operational_audit_report.md` | Operational audit findings |
| `audit_improvement_tracker_2026-06-27.md` | Tracked follow-ups from audit findings |

New audits land here as dated files. Security audits stay under `strategy/security/`.
