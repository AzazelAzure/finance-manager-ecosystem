# Strategy Automations

Prompt documents for recurring Antigravity (agy) automation runs.

## Prompts

| File | Schedule | Output |
|---|---|---|
| `prompts/daily_doc_sweep_prompt.md` | Daily — before daily summary | Plan registry reconciliation, `current_status.md` §5, `design_docs/` stale ref fixes, sweep report in `reports/` |
| `prompts/daily_summary_prompt.md` | Daily — end of workday | `strategy/daily_summaries/daily_summary_YYYY-MM-DD.md` + updates `strategy/current_status.md` |
| `prompts/legal_compliance_monthly_prompt.md` | Monthly (1st–3rd of month) | `strategy/legal/compliance_reports/YYYY-MM.md` |

## Execution order (daily)

1. **`scripts/gather_doc_context.sh`** — run first (local cron on HitM's machine, ~15 min before sweep). Gathers git log, PR lists, stale pattern scan, VPS state into `context/daily_context.md`.
2. **`daily_doc_sweep_prompt.md`** — Antigravity reads `context/daily_context.md`, reconciles plans, fixes stale refs, writes sweep report.
3. **`daily_summary_prompt.md`** — Antigravity generates the human-readable daily digest. Reads accurate plan state from the freshly-updated `plan_registry.md`.

## Directories

| Directory | Purpose |
|---|---|
| `prompts/` | Automation prompt files (source of truth for what each automation does) |
| `context/` | Generated context files from bash data-gather script (gitignored) |
| `reports/` | Sweep reports generated by daily_doc_sweep automation |

## How to run manually

```bash
# Generate context file (prerequisite for doc sweep)
./scripts/gather_doc_context.sh

# Daily doc sweep — paste prompt into Antigravity from repo root
cat strategy/automations/prompts/daily_doc_sweep_prompt.md

# Daily summary — paste prompt into Antigravity from repo root
cat strategy/automations/prompts/daily_summary_prompt.md

# Monthly legal compliance
cat strategy/automations/prompts/legal_compliance_monthly_prompt.md
```

## Cron setup (local machine)

See `scripts/gather_doc_context.sh` header for the exact crontab entry.
Run `crontab -e` and add the line from the script comments.
