# PH Hiring Roles and Wage Bands (Budget-Constrained)

Purpose: define realistic PH hiring options under current business affordability constraints, with staged role design and wage bands that avoid overcommitting cash flow.

Date seeded: 2026-05-05  
Status: pre-huddle planning artifact (for TP24 discussion).

## Context and constraint

- Standard PH market rates for technical roles can exceed current budget tolerance.
- This artifact prioritizes **survivable hiring** over market-max compensation.
- Roles are structured to start narrow and expand only after gate triggers are met.

## Hiring model principle

Use a staged path:
1. part-time or scoped contract first
2. validate output against measurable deliverables
3. expand scope/pay only after clear ROI or revenue gate

## Budget-constrained monthly wage bands (PHP)

### Tier 0 - Micro support (very constrained)

- **Range:** `10,000-20,000`
- **Structure:** part-time or task-bucket contract
- **Fit:** repetitive admin tasks, lightweight moderation, checklist execution
- **Not fit for:** engineering ownership, runtime responsibility, automation architecture

### Tier 1 - Ops assistant (entry feasible)

- **Range:** `20,000-35,000`
- **Structure:** part-time to near-full-time depending on scope
- **Fit:** admin operations, documentation upkeep, structured process support, SMM support
- **Risk:** requires strict scope boundaries to avoid role overflow

### Tier 2 - Technical operations coordinator

- **Range:** `35,000-55,000`
- **Structure:** full-time junior or strong part-time specialist
- **Fit:** orchestration support, agent runbooks, QA handoffs, basic automation operation
- **Risk:** limited ability for deep debugging or major system redesign

### Tier 3 - Junior engineer / orchestrator hybrid (budget edge)

- **Range:** `55,000-75,000`
- **Structure:** full-time or high-intensity contract
- **Fit:** meaningful technical ownership for selected slices, code+ops crossover
- **Risk:** likely upper edge of affordability under current constraints

## Suggested role options for current stage

### Option A - Admin/Ops first hire (lowest risk)

- **Band:** `20,000-35,000`
- **Primary outcomes:** reduce HitM coordination overhead, stabilize recurring processes
- **Upgrade trigger:** sustained throughput improvement for 6-8 weeks

### Option B - SMM execution hire (mobile-primary scope)

- **Band:** `18,000-35,000`
- **Primary outcomes:** content cadence + engagement handling
- **Condition:** desktop escalation path available for analytics/compliance tasks

### Option C - Technical ops/orchestration support hire

- **Band:** `35,000-55,000`
- **Primary outcomes:** runbook execution, pipeline monitoring, structured handoffs
- **Condition:** no assumption of senior engineering ownership at this band

### Option D - Junior engineering/orchestrator blended hire

- **Band:** `55,000-75,000`
- **Primary outcomes:** scoped technical implementation plus orchestration responsibilities
- **Condition:** only feasible with explicit budget gate and milestone-linked retention logic

## "AI Orchestrator" budget-safe framing

For current budget reality, avoid pricing this as a pure AI engineer role.

Use two internal levels:
- **Assistant Orchestrator:** `30,000-45,000` (process execution, documentation discipline, agent handoff hygiene)
- **Orchestrator (technical):** `45,000-65,000` (adds limited technical debugging and automation operation)

Above `65,000` should require explicit budget and output gates.

## Employment structure guidance

### Part-time first (recommended default)

- lower cash risk
- easier scope correction
- clearer performance signal before full-time conversion

### Contract packets for technical tasks

- define deliverables by slice/task packet
- pay by milestone acceptance
- reduces open-ended payroll risk during uncertain throughput periods

### Full-time only after gate pass

Gate examples:
- stable monthly revenue threshold
- delivery throughput sustained over 2 consecutive cycles
- no recurring missed obligations from current baseline workload

## Hidden employer-cost reminder

Budget planning should include total employer cost, not salary alone:
- mandatory PH employer obligations where applicable
- equipment/procurement and setup costs
- tooling/subscription costs
- onboarding/training time cost

## Recommended default sequence (if hiring starts soon)

1. start with **Tier 1 or Tier 2** role
2. prove measurable capacity gain
3. move to Tier 3 only after gate confirmation
4. defer senior engineering pay bands until post-S1.B or stronger revenue footing
