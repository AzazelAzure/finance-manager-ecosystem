# T02.SL2: TourProvider Persistence Wire

## Scope
- **Surface**: Web `TourProvider`
- **Focus**: Connecting tour completion to the AppProfile API.

## Execution Checklist
- [ ] Verify `markTourCompleted` is invoked when a tour finishes or is skipped.
- [ ] Ensure `markTourCompleted` calls `updateProfile({ completed_tours: [...completedTours, tourId] })`.
- [ ] Ensure that upon success, the `['app-profile']` react-query cache is invalidated to refetch the new state.

## Verification
- Observe network tabs during a tour finish event. The PATCH request should contain the updated array.
- Subsequent calls to `isTourCompleted(tourId)` should return true.
