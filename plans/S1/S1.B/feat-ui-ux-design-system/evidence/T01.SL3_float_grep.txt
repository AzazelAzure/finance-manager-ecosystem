## DEVIATION FROM TASK SPEC (flagged for plan owner)
T01.SL3 acceptance asks to install currency.js/Dinero.js wherever monetary
floats are found. NOT done in this slice, deliberately:
  - The genuine float math lives in the OFFLINE path (outbox/calendar/viz
    overlays). Migrating that to currency.js is a behavior-bearing change to
    PWA sync logic, which this plan must NOT regress (PWA class B), and it
    needs its own test surface.
  - Bundling that migration into a foundational tokens PR violates
    "one coherent intent per change set" and risks the offline contract.
RECOMMENDATION: open a dedicated follow-up slice (e.g. T01.SL3b or a separate
fix plan) to introduce integer-centavo money handling (currency.js) across
the three offline overlay modules, with offline-sync regression tests.
The display path (formatMoney) can adopt the same helper at that time.
