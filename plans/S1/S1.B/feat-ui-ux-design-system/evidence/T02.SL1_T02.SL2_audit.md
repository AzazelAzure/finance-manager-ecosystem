# T02 — Component Primitive Audit + Token Wiring

**Branch:** `cur/s1b/feat/ui-ux-design-system`  
**Date:** 2026-06-26

## Audit Result

- `package.json` has no `shadcn` / `@shadcn/ui` dependency.
- No `components.json` exists.
- The app has an established custom primitive layer in `src/components/ui/`
  (`Button`, `Card`, `Modal`, `KPI`, `DataTable`, etc.) backed by
  `src/components/ui/ui.css`.
- The repo does not use Tailwind; running `npx shadcn init` would introduce a
  new styling stack and risk overwriting custom primitives.

## Action Taken

- Kept the existing custom primitive system.
- Added `src/components/ui/Input.tsx` as a first-class primitive for the
  existing `.ui-input` tokenized class.
- Rewired `Card`, `Button`, `Input` CSS paths to reference the Stage 0
  `--color-*` tokens:
  - `--color-surface-border`
  - `--color-surface-card`
  - `--color-brand-primary`

## Verification

- `npm run build`: PASS.
- `npm run lint`: FAILS on pre-existing baseline issues outside this T02 work
  (API `any`, F-007 tour fast-refresh exports, offline connectivity `any`,
  existing DashboardPage tour `as any`, etc.).

## Deviation

T02 is implemented as a **shadcn-compatible audit/token wiring** pass, not a
shadcn scaffold. This is intentional because the repo has no Tailwind and
already owns a custom primitive layer.
